# RevoFinance Personal Finance & Spending Analysis

## SECTION 1: PROJECT SUMMARY FOR PORTFOLIO
### Summary / Context
This multi-phase analytics project explores operational optimization across three fictional companies: **RevoGrocers (Retail), RevoBank (Banking), and RevoFinance (Fintech)**.
The objective is to transform raw transaction data into actionable insights using **SQL analysis, Python machine learning, and financial data visualization**.
The RevoFinance component focuses on building dashboards that help users **track spending patterns, monitor budget utilization, and improve personal financial management.**

### Goals
The project aims to:
• Identify high-revenue product categories in RevoGrocers using SQL  
• Segment RevoBank customers based on financial risk and value using Python clustering  
• Build financial dashboards for RevoFinance to help users monitor spending behavior and budget utilization  

### Process
The project combines multiple analytical approaches:
1. SQL analysis to extract key sales insights and product performance  
2. Root Cause Analysis (RCA) to diagnose operational inefficiencies  
3. Python machine learning using **K-Means clustering** for customer segmentation  
4. Financial data visualization dashboards to monitor spending patterns and budget consumption  

The **OBIPR framework** was applied to structure insights:
Observation → Business Impact → Insight → Proposal → Result

### Output
Key insights from the analysis include:
• Confections identified as the top revenue driver generating **$556.93 Million**  
• RevoBank users segmented into **three customer clusters**, identifying high-value low-risk users  
• RevoFinance data revealed a **strong offline spending concentration in Jakarta**  
• A significant **end-of-year spending spike** was detected across multiple merchant categories  

Based on these insights, I recommended:
• Seasonal budgeting alerts  
• Merchant partnership optimization  
• Spending monitoring dashboards to improve financial health
